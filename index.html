Selamat Datang di Mini Inventory

Aplikasi ini dibangun menggunakan APP Forge — sebuah code generator ringkas yang mengubah konfigurasi JSON menjadi aplikasi web lengkap dengan antarmuka profesional, autentikasi, dan operasi CRUD.

Filosofi APP Forge

APP Forge lahir dari sebuah pemikiran sederhana: "Bagaimana jika pembuatan aplikasi web tidak dimulai dari menulis kode, melainkan dari mendefinisikan kebutuhan?"

Dalam paradigma tradisional, developer memulai project dengan setup lingkungan, menulis HTML/CSS/JavaScript satu per satu, copy-paste boilerplate yang sama berulang kali, dan menghabiskan waktu untuk hal-hal repetitif yang tidak memberikan nilai bisnis.

APP Forge membalik paradigma ini. Dengan pendekatan Configuration-over-Code, seluruh aplikasi web didefinisikan melalui satu file JSON payload yang mendeskripsikan struktur data, perilaku UI, navigasi, permission, hingga workflow bisnis.

Satu file konfigurasi JSON menghasilkan seluruh aplikasi yang siap jalan — lengkap dengan CRUD, autentikasi, validasi form, DataTable, dan antarmuka profesional berbasis Bootstrap 5.

Fitur Utama

Kemampuan inti yang menjadikan APP Forge sebagai platform code generation yang lengkap.

JSON-Driven Development

Seluruh aplikasi didefinisikan dalam satu file JSON. Struktur data, field, validasi, tabel, form, filter — semua dikonfigurasi secara deklaratif tanpa menulis kode HTML atau JavaScript secara manual.

Plugin Architecture

UI framework dan tema dapat diganti tanpa mengubah konfigurasi aplikasi. Satu payload JSON yang sama dapat menghasilkan output yang berbeda hanya dengan mengganti plugin yang digunakan.

JWT Authentication

Sistem login dengan JWT token terintegrasi secara built-in. Mendukung token refresh, idle timeout otomatis, dan permission-based access control untuk setiap halaman.

Smart DataTable

Tabel data dengan sorting, searching, dan pagination secara otomatis. Mendukung status filter, date range filter, inline action, serta konfigurasi kolom yang fleksibel.

Master-Detail Pattern

Mendukung pola header-detail (parent-child) dengan inline editing pada tabel detail. Cocok untuk invoice, purchase order, dan transaksi lain yang memiliki baris item.

Scoped & Incremental

Generate satu halaman form tanpa mengganggu halaman lain menggunakan mode scoped generation. Manifest tracking memastikan setiap perubahan tercatat untuk update parsial yang aman.

Keunggulan APP Forge vs Pendekatan Tradisional

Perbandingan proses pengembangan aplikasi web antara pendekatan tradisional (menulis kode manual) dengan paradigma APP Forge (configuration-driven generation).

Aspek Pendekatan Tradisional APP Forge
Setup Project Awal Memerlukan waktu berjam-jam hingga berhari-hari untuk konfigurasi build tool, folder structure, routing, dan boilerplate code. Satu perintah appforge init menghasilkan project lengkap dalam hitungan detik, siap dikembangkan.
Konsistensi UI Bergantung pada disiplin developer. Setiap halaman berpotensi memiliki pola UI dan penamaan yang berbeda-beda. Dijamin konsisten karena semua halaman dihasilkan dari template yang sama dengan konvensi yang seragam.
Boilerplate Code Copy-paste berulang untuk setiap halaman baru: form HTML, validasi JS, AJAX call, DataTable initialization. Zero boilerplate. Setiap halaman baru cukup ditambahkan sebagai entry baru di JSON payload.
Autentikasi Implementasi manual: login form, token management, session handling, permission check di setiap halaman. Built-in JWT authentication dengan token refresh, idle timeout, dan permission-based access control.
Operasi CRUD Menulis kode CRUD (Create, Read, Update, Delete) secara manual untuk setiap entitas/halaman. Otomatis. Cukup definisikan field, primaryKey, dan apiPath — CRUD lengkap langsung tersedia.
Perubahan & Update Edit file satu per satu. Risiko inkonsistensi tinggi jika ada perubahan struktur yang melibatkan banyak file. Ubah konfigurasi JSON, lalu re-generate. Scoped generation memungkinkan update satu halaman saja.
Onboarding Developer Developer baru perlu memahami keseluruhan codebase, konvensi tim, dan pola arsitektur yang digunakan. Cukup pahami struktur JSON payload. Konvensi dan arsitektur sudah ditentukan oleh plugin.
